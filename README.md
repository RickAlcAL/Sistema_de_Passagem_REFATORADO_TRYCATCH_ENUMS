# Ticketing Booking Refactored

Este projeto apresenta a refatoração de um **Sistema de Emissão de Passagens**. A arquitetura foi aprimorada para garantir a segurança no momento da reserva de assentos e consistência na escolha de classes de viagem.

## 🚀 Melhorias Aplicadas

* **Tratamento de Exceções (`try-catch`):** Validação de disponibilidade de poltronas, checagem de dados do passageiro e tratamento de falhas em transações financeiras.
* **Uso de `Enums`:** Definição clara das classes de viagem (ex: `ECONOMICA`, `EXECUTIVA`, `PRIMEIRA_CLASSE`) e status da passagem (ex: `RESERVADA`, `CONFIRMADA`, `CANCELADA`).

## 🛠️ Tecnologias Utilizadas

* **Linguagem:** Java
* **Paradigma:** Orientação a Objetos (POO)
